面试题：ORMLite数据库使用流程及场景

在项目开发中，我们常常会使用到数据库，一般数据库使用场景用于存储。在Android中主要使用SQLite数据库，但SQLite难免有些繁琐，所以我们采用ORMLite，简化使用。

简介

ORM它的英文全称是Object Relational Mapping，是对象关系映射（Object Relational Mapping）的缩写，对象和关系数据是业务实体的两种表现形式，业务实体在内存中表现为对象，在数据库中表现为关系数据，ORM实现了对象和关系数据库之间的转换，简单来说，就是我们定义一个实体类，利用这个框架，它可以帮我们把这个实体映射到我们的数据库中，对应一个数据库表。

使用过程

1. 添加依赖

根据ORMLite的官方使用说明，一般需要添加两个依赖：

compile 'com.j256.ormlite:ormlite-core:5.0' // 这个包提供了JDBC和Android的核心功能包
compile 'com.j256.ormlite:ormlite-android:5.0' // 这个包提供了Android的特定功能

2. 定义数据库的帮助类

帮助类继承自OrmLiteSqliteOpenHelper，主要是帮助我们进行数据库表的创建和更新。

// 继承自OrmLiteSqliteOpenHelper
public class DBHelp extends OrmLiteSqliteOpenHelper{

    // 数据库的名称
    private static final String DB_NAME="repo_favorite.db";

    // 数据库版本
    private static final int VERSION = 2;

    private static DBHelp dbHelp;
    private Context context;

    // 实现单例化
    public static synchronized DBHelp getInstance(Context context){
        if (dbHelp==null){
            dbHelp = new DBHelp(context.getApplicationContext());
        }
        return dbHelp;
    }

    private DBHelp(Context context) {
        super(context, DB_NAME, null, VERSION);
        this.context = context;
    }

    // onCreate方法里面会进行数据库的创建，在这里面我们同时可以实现数据库表的创建及数据库表的数据填充
    @Override
    public void onCreate(SQLiteDatabase database, ConnectionSource connectionSource) {

    }

    // 对数据库的表进行更新，一般常用方法就是先删除，再创建。
    @Override
    public void onUpgrade(SQLiteDatabase database, ConnectionSource connectionSource, int oldVersion, int newVersion) {

    }
}

3. 创建数据库表

在Android中利用ORMLite框架创建数据库表，是创建一个实体类，利用注解的方式表明其为数据库中的一张表，实体类里面的所有成员变量都可以设置为表中的字段，也是利用注解的方式来表明的。举个栗子：

// 在实体类的上面利用注解表明他是我们数据库里面的一张表，并设置表名。
@DatabaseTable(tableName = "repostiory_group")
public class RepoGroup {

    // 这是个主键
    @DatabaseField(id = true)
    private int id;

    @DatabaseField(columnName = "NAME")
    private String name;

    @Override
    public String toString() {
        return "RepoGroup{" +
                "id=" + id +
                ", name='" + name + '\'' +
                '}';
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

常用注解说明：

  1. @DatabaseTable(tableName = "local_repo")在实体类上面定义，表明这个实体类是数据库的一张表，并且可以利用tableName设置表名
  2. @DatabaseField 表明这个变量是数据库表里面的一个字段。里面涉及到几个内容：

id=true表明这个是一个主键，如果指定为generatedId，表示自动增长id，我们不需要给它赋值。

columnName=""这个表示我们为其字段指定一个字段名，一般不指定的时候就是默认的变量名。

foreign = true表明是一个外键，就是关联了数据库中另外的一张表，这个字段的信息就是另外一张表里面的某一条信息。

4. 定义数据库表的操作类

数据库和表都有了之后，定义一个操作类DAO，对数据库里面的表进行操作：添、删、改、查的操作，需要注意的是，是定义一个表的操作类，所以表和表的操作类是一一对应的关系。

// 某一表的操作类
public class RepoGroupDao {

// 通过之前定义的数据库的帮助类来帮我们完成操作类的实例化
private Dao<LocalRepo,Long> dao;
public LocalRepoDao(DBHelp dbHelp) {
    try {
        dao = dbHelp.getDao(LocalRepo.class);
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 添加和更新单条数据
 */
public void createOrUpdate(LocalRepo localRepo){
    try {
        dao.createOrUpdate(localRepo);
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 添加和更新多条数据
 */
public void createOrUpdate(List<LocalRepo> list){
    for (LocalRepo localrepo:list) {
        createOrUpdate(localrepo);
    }
}

/**
 * 删除某一条数据
 */
public void delete(LocalRepo localrepo){
    try {
        dao.delete(localrepo);
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 修改某一条数据
 */
public void update(LocalRepo localrepo){
    try {
        dao.update(localrepo);
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 查询---根据条件查询（id查询）
 */
public List<LocalRepo> queryForGroupId(int groupId){
    try {
        return dao.queryForEq(LocalRepo.COLUMN_GROUP_ID,groupId);
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 查询---根据条件查询：利用where()方法来自定义查询
 */
public List<LocalRepo> queryNoGroup(){
    try {
        return dao.queryBuilder().where().isNull(LocalRepo.COLUMN_GROUP_ID).query();
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

/**
 * 查询所有的数据
 */
public List<LocalRepo> queryAll(){
    try {
        return dao.queryForAll();
    } catch (SQLException e) {
        throw new RuntimeException(e);
    }
}

上述代码中就是常用的增删改查的操作。

5. 使用

初始化DBHelp帮助类和DAO的操作类，即可以实现根据数据对数据库进行操作。

使用总结及场景

使用流程总结及注意事项：

  1. 添加依赖
  2. 数据库帮助类：帮我们完成数据库的创建及更新，注意点：数据库中做更新之后，不更改版本号，不会自动做数据库更新。
  3. 数据库表：因为是对象关系映射，所以创建实体类作为表，注意点：一定要注解指定它是数据库表。
  4. 数据库表操作类：对数据库表实现添删改查的操作，注意点：一个操作类Dao对应一个表，仅对一个表进行操作，是一对一的关系。
